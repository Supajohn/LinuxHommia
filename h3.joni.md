Ansiblella voidaan automatisoida asennus ja konfigurointi helpommin Apache2 tyylillä. Siinä saa saman lopputuloksen joka ajolla jos oikein tehtynä.

Handlerit toimii "muutoksen" ilmoittajana. Eli handlers ajetaan vain jos jokin tehtävä ilmoittaa muutoksesta. Eli helpottaa automatisaatiota sillä se ajaa uudelleen käynnistyksen vain jos config on muuttunut.

Notify toimii koodina jolla se herättää tämän muutoksen ilmoittajan "Handlers". Handler suoritetaan usein vasta ansible playbook komennon lopussa, tai kun config on muuttunut.

Ansible-doc service (moduuli)
Enabled tarkoittaa käynnistyykö suoraan bootissa.
Name antaa oikein systemd palvelun kuten nginx tai apache2 lataus.
State määrittää ja ohjaa palvelun tilaa eli started/stopped.
Examples kertoo ja auttaa ymmärtämään syntakseja nopeammin.

a) Apache2 käsin tehtynä. 

Komento sudo apt-get install apache2 lataa apache2 paketin.
Apache2 pitää käynnistää joten systemctl start apache2

Selaimella avautuessaan näkyy suoraan apache2 default sivusto.
